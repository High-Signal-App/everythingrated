# Deploy

EverythingRated deploys as a single Cloudflare Worker (`everythingrated`)
hosting the whole Next.js app plus static assets, backed by one D1 database
(`everythingrated-db`). Production origin: <https://ratings.highsignal.app>.

The deploy is **manual only** — it is not wired to push. The
`cloudflare-deploy.yml` workflow runs on `workflow_dispatch`. Do not enable
auto-deploy without revisiting the migration-first ordering below.

## Prerequisites

- `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID` in the deploy
  environment (GitHub secret for the workflow; `npx wrangler login` for
  local). Never commit these.
- The remote D1 schema must be at or ahead of the code's expected schema
  (see [runbooks/d1-migrations.md](runbooks/d1-migrations.md)).

## Order of operations

Migrations before code, always. A deploy that ships code expecting a column
the remote DB does not have will 5xx.

1. **Pull / build locally and verify:**
   ```bash
   pnpm install --frozen-lockfile
   pnpm typecheck
   pnpm test
   pnpm cf:build
   ```
2. **Apply pending migrations to remote D1:**
   ```bash
   pnpm db:migrate:remote
   ```
   Inspect with `wrangler d1 execute everythingrated-db --remote \
   --command "SELECT name FROM d1_migrations ORDER BY id" \
   --config=apps/web/wrangler.toml` if anything looks off.
3. **Back up before destructive migrations** — see the runbook. There is no
   automated backup; run `wrangler d1 export` first.
4. **Deploy the Worker:**
   ```bash
   pnpm deploy        # opennextjs-cloudflare build && deploy
   ```
   Or via the GitHub Actions `Deploy Cloudflare` workflow (manual dispatch).
5. **Smoke test** the production origin:
   ```bash
   curl --fail --show-error --silent --location --max-time 30 \
     https://ratings.highsignal.app/ > /dev/null
   ```
   The `cloudflare-deploy.yml` workflow does this automatically after deploy.

## What the deploy ships

- `apps/web/worker.mjs` — Worker entry: `handleAgentEdge` → edge-cache check
  → `withTiming(openNext.fetch)` → global try/catch. See
  [architecture/edge-and-agent-surfaces.md](../architecture/edge-and-agent-surfaces.md).
- `.open-next/` — OpenNext build output (gitignored, generated by
  `pnpm cf:build`).
- `apps/web/.open-next/assets/` — static assets bound as `ASSETS`.
- D1 binding `DB` → `everythingrated-db` (declared in `apps/web/wrangler.toml`).

## Worker config that affects production

From `apps/web/wrangler.toml`:

- `[placement] mode = "smart"` — runs the Worker near the D1 datacentre.
  psi-swarm flagged TTFB > 1s before this; do not remove it.
- `[observability] enabled = true, head_sampling_rate = 0.1` — 10% trace
  sampling.
- `[limits] cpu_ms = 30000` — 30s CPU limit.
- `compatibility_flags = ["nodejs_compat"]` — required by OpenNext.

## Remote seed

`pnpm db:seed:remote` writes owner ratings and catalogue rows to production.
**Never run it casually** — it is a deliberate, coordinated action, not a
deploy step. See [development/seed-data.md](../development/seed-data.md).

## Rollback

Workers are immutable deploys — `wrangler deployments rollback` returns the
previous version. D1 migrations are forward-only (no transactional rollback);
undo via a new compensating migration or a `wrangler d1 export` restore. See
[runbooks/d1-migrations.md](runbooks/d1-migrations.md).
