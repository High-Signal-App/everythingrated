# Code conventions

## Formatting & linting

- **Biome** is the formatter and linter (`biome.json`): 2-space indent,
  100-col width, single quotes, semicolons, LF, ES5 trailing commas, double
  JSX quotes.
- `pnpm check` runs `biome check .`; `pnpm format` writes.
- Biome ignores generated/build dirs and `packages/db/migrations/**` (do not
  hand-format migrations).
- `apps/web`'s `lint` script is a no-op echo — Next 16 deprecated
  `next lint`. Use Biome. ESLint config (`eslint.config.mjs`) remains for
  editor integration via `eslint-config-next`.

## TypeScript

- Strict mode, `target: ES2022`, `moduleResolution: Bundler`, `isolatedModules`.
- `noEmit` — typecheck only; Next.js / OpenNext handle emission.
- `pnpm typecheck` runs `tsc --noEmit` across workspaces.

## Project layout

- `apps/web/src/app/` — Next.js App Router pages and route handlers.
- `apps/web/src/components/` — atomic design: `atoms/`, `molecules/`,
  `organisms/`. Plus `posthog-provider.tsx`, `saasmaker-feedback.tsx`,
  `VitalsReporter.tsx`.
- `apps/web/src/lib/` — DB access, rating queries, server actions, visitor
  cookie, moderation, analytics, stack recommender, RSS, vitals. Tests live
  next to the module (`*.test.ts`).
- `packages/db/src/` — Drizzle schema + D1 client. Migrations in
  `packages/db/migrations/`.

## Server Components vs Server Actions

- **Reads** go in Server Components / Route Handlers via `getDb()` and the
  `lib/ratings.ts` query helpers. Never mint cookies here.
- **Writes** go in Server Actions (`lib/actions.ts`) via `ensureVisitorId()`.
  Always call `revalidatePath` after a mutation so the authoritative view
  refreshes.
- **No middleware.** Visitor ids are minted only in mutating Server Actions
  to keep page renders cacheable and bot-noise-free. See
  [ADR-0001](../architecture/decisions/ADR-0001-no-auth-in-poc.md).

## Tests next to code

Vitest picks up `apps/web/src/**/*.test.ts`. Co-locate tests with the module
they exercise (e.g. `ratings.ts` ↔ `ratings.test.ts` is the pattern, though
current tests are named by feature: `collections.test.ts`,
`comparison.test.ts`, `directory-submissions.test.ts`,
`item-submissions.test.ts`, `query-intent.test.ts`, `stack-recommender.test.ts`).

## Comments

- Do not add or remove comments unless asked. If you accidentally delete a
  comment, restore it.
- The schema file uses comments to explain *why* a column or index exists
  (e.g. the `superseded` partial-style index). Preserve those — they encode
  non-obvious hot-path reasoning.

## Generated files — do not hand-edit

- `apps/web/agent-edge.mjs` — regenerated by the fleet `apply-agent-surfaces`
  tool. Recent commits reconcile generator output with Biome formatting;
  keep that reconciliation when regenerating.
- `packages/db/migrations/*.sql` — drizzle-kit output. Edit
  `packages/db/src/schema.ts` and run `pnpm db:generate`.
- `apps/web/public/llms.txt`, `llms-full.txt`, `index.md`, `api-ai.json` —
  static fallbacks for the agent surfaces; the edge handler is the live
  source.

## What not to add (POC constraints)

Auth, search, item submission beyond the `ai-dev-tools` pilot, comments,
admin panel beyond `/moderation`, email, dynamic directory creation, broad
public submission without moderation. See
[product/overview.md](../product/overview.md) and
[ADR-0001](../architecture/decisions/ADR-0001-no-auth-in-poc.md).
