# Worker entry, edge caching, and agent surfaces

## Worker entry (`apps/web/worker.mjs`)

The Worker entry wraps OpenNext with two middlewares and re-exports Durable
Object bindings so wrangler can resolve them at deploy time:

```ts
import openNext from "./.open-next/worker.js";
import { withTiming } from "./timing.mjs";
import { handleAgentEdge } from './agent-edge.mjs';

export { DOQueueHandler, DOShardedTagCache, BucketCachePurge }
  from "./.open-next/worker.js";
```

Request order:

1. `handleAgentEdge(request)` — short-circuits agent indexing surfaces.
2. Edge-cache check for cacheable exact paths (see below).
3. `withTiming(openNext.fetch)` — measures duration, sets `Server-Timing`,
   logs slow (>200ms) requests via `console.warn`.
4. Global try/catch error handler (added 2026-07-02) returns a 500 instead
   of crashing the Worker on uncaught throws.

## Edge caching

A `CACHEABLE_EXACT` set of paths (`/`, `/about`, `/trending`, `/list`,
`/stack`, `/random`, `/aspects`, `/api-docs`, …) gets a
`public, max-age=3600, s-maxage=86400, stale-while-revalidate=604800`
`Cache-Control`. This is what makes marketing/value-add pages serve in
sub-second TTFB. Do not add dynamic or per-visitor pages to this set.

## Agent indexing surfaces

`apps/web/agent-edge.mjs` is a **generated** portable handler (spec:
`fleet-ops/docs/agent-indexing-standard.md`). It serves:

- `/llms.txt` — short agent index.
- `/llms-full.txt` — full agent brief.
- `/index.md` — homepage markdown (product brief without JS).
- `/api/ai` — JSON catalog of public surfaces.
- `/sitemap.xml`, `/robots.txt` (also via Next.js `sitemap.ts` / `robots.ts`).

The `AGENT_SURFACE` payload in `agent-edge.mjs` is regenerated by the fleet
`apply-agent-surfaces` tool — do not hand-edit it; regenerate. The
`apps/web/public/` copies (`llms.txt`, `llms-full.txt`, `index.md`,
`api-ai.json`) are static fallbacks; the edge handler is the live source.

## Feeds

- `/d/[directory]/items.json` — per-directory item inventory.
- `/d/[directory]/rss` — per-directory RSS.
- `/d/[directory]/[item]/item.json` — single-item JSON.
- `/directories.json` — all directories.
- Item pages advertise feeds via `<link rel="alternate">` and include rating
  JSON-LD. `/feeds` is a human-readable feed index.

## Sitemap & robots

- `apps/web/src/app/sitemap.ts` — `force-dynamic`, `revalidate = 3600`.
  Enumerates homepage, marketing/value-add pages, focused directory, items,
  aspects. Parked directories are excluded from the sitemap.
- `apps/web/src/app/robots.ts` — allows `/`, disallows `/api/`, `/moderation`,
  `/submit-directory`. Sitemap points at the production origin
  `https://ratings.highsignal.app`.
- `IndexNow` key file at `apps/web/public/fa7259e2e0d942f1a1267b344a75a143.txt`
  for Bing/Yandex URL submission.

## Well-known

- `apps/web/public/.well-known/security.txt`.
- `apps/web/src/app/humans.txt/route.ts`.
- `apps/web/src/app/manifest.ts` — PWA manifest.

## Adding a new agent surface

1. Update the fleet `apply-agent-surfaces` source (not this repo).
2. Regenerate `agent-edge.mjs` and the `public/` static copies.
3. Commit the regenerated `agent-edge.mjs` (recent commits reconcile
   generator output with Biome formatting — keep that reconciliation).
